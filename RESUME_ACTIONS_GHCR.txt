RESUME DES ACTIONS - GHCR ET PROJET MLOPS
=========================================

Decision actuelle :
- Le registry principal est GitHub Container Registry (GHCR).
- Image finale : ghcr.io/vina771/projet11-mlops:latest
- Le scan de securite reste assure par Trivy dans le pipeline GitLab CI/CD.
- Le deploiement reste assure par Ansible.

Ce qui a ete nettoye dans le projet :
- Pipeline GitLab CI/CD en 6 stages conserve.
- Variables registry migrees vers REGISTRY, IMAGE_NAME, GHCR_USER, GHCR_TOKEN.
- Stage build : tag de l'image GHCR avec latest et CI_PIPELINE_ID.
- Stage scan : Trivy scanne l'image GHCR locale avant push.
- Stage push : docker login ghcr.io puis push des deux tags.
- Stage deploy : Ansible pull l'image depuis GHCR.

Variables GitLab CI/CD a configurer :
- REGISTRY=ghcr.io
- IMAGE_NAME=ghcr.io/vina771/projet11-mlops
- GHCR_USER=Vina771
- GHCR_TOKEN=(GitHub PAT packages, masked/protected)

Token GitHub requis :
- write:packages
- read:packages

A verifier ensuite :
- Runner GitLab enregistre et actif.
- Variables CI/CD bien creees dans GitLab.
- Pipeline complet vert.
- Image visible dans GitHub -> Packages.
- Application locale accessible sur http://localhost:8501.

FastAPI est maintenant utilisee directement par Streamlit :

- En Docker Compose, Streamlit appelle `http://fastapi:8000`.
- Une page `Tester API FastAPI` permet de tester `/health`, `/metrics`, `/predict` et `/predict/batch`.

Pipeline :
- Build image Streamlit : `ghcr.io/vina771/projet11-mlops`.
- Build image FastAPI : `ghcr.io/vina771/projet11-fastapi`.
- Trivy scanne les deux images.
- Le stage push pousse les deux images avec les tags `$CI_PIPELINE_ID` et `latest`.
